# NEXIO 신뢰 시스템 완성 계획서 (v7.1)

> **상태**: 기획 완료 (Phase 1)
> **대상**: NEXIO 자동매매 시스템 v7.0+
> **목표**: 운영 안정성 강화, 보안 취약점 제거, 자가 진단 및 복구 체계 구축

## 1. 개요
NEXIO v7.0은 핵심 매매 로직과 학습 엔진이 완성된 상태입니다. 그러나 실제 운영 환경에서 발생할 수 있는 환경변수 누락, API 노출 리스크, 엔진 중단 상황에 대한 대응 체계가 부족합니다. 본 계획서는 "신뢰할 수 있는 시스템"을 위해 보안과 안정성이라는 마지막 퍼즐을 맞추는 작업을 정의합니다.

## 2. 사용자 스토리
- **운영자**: "환경변수가 하나라도 빠지면 엔진 실행 전에 즉시 알림을 받고 싶다."
- **개발자**: "로그에 KIS AppSecret이 노출되지 않도록 URL 쿼리 파라미터를 제거하고 싶다."
- **사용자**: "엔진이 네트워크 오류로 잠시 멈추더라도 다음 실행 때 스스로 상태를 복구해서 정상 작동하길 원한다."
- **검수자**: "핵심 경로(로그인, 잔고 조회, 엔진 상태)가 정상인지 자동화된 테스트로 매일 확인하고 싶다."

## 3. 상세 기능 및 기술 요구사항

### 3.1. balance API POST 전환 (보안 강화)
- **현황**: `/api/kis/balance` 호출 시 `appSecret` 등 민감 정보가 URL 쿼리 스트링으로 전달됨.
- **개선**:
    - `src/app/api/kis/balance/route.ts`의 `GET` 메서드를 `POST`로 변경.
    - 요청 바디(JSON)에서 자격 증명을 수신하도록 수정.
    - `src/lib/kis/client.ts`의 `fetchBalance` 및 `fetchPrice` 함수를 `POST` 호출 방식으로 업데이트.
    - (선택 사항) `/api/kis/price` 등 다른 KIS 프록시 API도 동일하게 보안 강화.

### 3.2. 엔진 자가복구 로직 (안정성 강화)
- **현황**: 엔진 실행 중 오류 발생 시 알림만 보낼 뿐, 중단된 시점의 상태(예: 미체결 주문)를 스스로 정리하지 않음.
- **개선**:
    - **중복 실행 방지**: `engine_runs` 기록을 조회하여 현재 이미 실행 중인 프로세스가 있는지 확인 (Liveness Check).
    - **고립 상태 정리 (Stale State Cleanup)**: 엔진 시작 시 `pending_orders` 중 생성된 지 일정 시간(예: 30분)이 지난 오더를 취소 처리하거나 만료 처리.
    - **재시도 래퍼 (Retry Wrapper)**: KIS API 토큰 발급 및 잔고 조회 등 핵심 네트워크 호출에 대해 최대 3회 지수 백오프(Exponential Backoff) 재시도 적용.

### 3.3. 환경변수 누락 자동 감지 (자가 진단)
- **현황**: 환경변수가 누락된 경우 런타임에 `undefined` 오류로 엔진이 비정상 종료됨.
- **개선**:
    - `src/lib/config-validator.ts` 신규 생성.
    - 필수 환경변수 리스트(SUPABASE, KIS, TELEGRAM, ADMIN 등) 검증 로직 구현.
    - 엔진 가동(`GET /api/engine`) 직후 첫 단계에서 검증 수행.
    - 누락 발생 시 텔레그램으로 "⚠️ 설정 오류: [변수명] 누락" 알림 발송 후 즉시 종료.

### 3.4. 핵심 경로 최소 E2E 테스트 (회귀 방지)
- **현황**: 코드 수정 후 수동으로 브라우저를 열어 확인해야 함.
- **개선**:
    - Playwright(`@playwright/test`) 도입.
    - `tests/core.spec.ts` 작성:
        1. **Login**: 관리자 로그인이 성공적으로 수행되는가?
        2. **Dashboard**: 잔고 및 포트폴리오 데이터가 정상적으로 렌더링되는가?
        3. **Settings**: KIS 설정 상태가 올바르게 표시되는가?
    - GitHub Actions 또는 로컬 스크립트로 주기적 실행 환경 마련.

## 4. 우선순위 및 일정

| 우선순위 | 작업 항목 | 중요도 | 예상 공수 |
| :--- | :--- | :--- | :--- |
| **P0** | 환경변수 누락 자동 감지 | 매우 높음 | 0.5d |
| **P0** | balance API POST 전환 | 매우 높음 | 0.5d |
| **P1** | 엔진 자가복구 (중복 실행 방지 및 정리) | 높음 | 1d |
| **P2** | 핵심 경로 E2E 테스트 구축 | 보통 | 1d |

## 5. 영향 범위
- **백엔드**: `/api/kis/*`, `/api/engine` 라우트 핸들러 수정.
- **프론트엔드**: `src/lib/kis/client.ts` 및 이를 사용하는 컴포넌트(`PortfolioTab`, `EngineControlSection`).
- **인프라**: Playwright 의존성 추가, 환경변수 관리 가이드 업데이트.
